## Findings

- **jspdf new critical CVEs**: The advisory database added 10 new jsPDF vulnerabilities (ReDoS, DoS, Local File Inclusion, PDF Injection, Stored XMP Injection, Race Condition). Fix requires `jspdf@4.2.0` but `jspdf-autotable` (used for payslip table generation) has no v4 release ‚Äî cannot upgrade without breaking payslips.
- **next new high CVEs**: Two new CVEs appeared for next@14.x (Image Optimizer remotePatterns DoS + RSC deserialization DoS). Fix requires next@16. Out of scope.
- **ai SDK v6 breaking changes**: The API changed substantially. Key changes made:
  - `streamText()` is now synchronous (no `await`)
  - `toDataStreamResponse()` renamed to `toTextStreamResponse()`
  - `useChat` import moved from `ai/react` ‚Üí `@ai-sdk/react`
  - `input/handleInputChange/handleSubmit/isLoading` replaced by `sendMessage/status` API
  - `message.content` replaced by `message.parts.find(p => p.type === "text")?.text`
  - Transport now configured via `TextStreamChatTransport` class
- **Buffer type issue (Node 22 + ExcelJS)**: `Buffer.from(arrayBuffer)` returns `Buffer<ArrayBuffer>` in Node 22 types, incompatible with ExcelJS `xlsx.load()` which expects old `Buffer`. Fixed by passing `arrayBuffer as ArrayBuffer` directly (ExcelJS accepts ArrayBuffer natively).
